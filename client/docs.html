Getting started

SoulEXE scripts are standard FiveM resources. Each download is a folder containing a fxmanifest.lua, client/server Lua files, and a README.md with any script-specific setup steps.

  1. Download the script from the library.
  2. Unzip it into your server's resources/ directory.
  3. Add ensure your-script-name to your server.cfg.
  4. Restart your server or run refresh then ensure your-script-name in the console.

Installing a script

resources/
└── [soulexe]/
    └── se-garage/
        ├── fxmanifest.lua
        ├── client/
        ├── server/
        ├── config.lua
        └── README.md

We recommend grouping SoulEXE resources under a [soulexe] category folder to keep your resources directory tidy — FiveM ignores brackets in folder names when resolving resource names.

Configuration

Every script ships with a top-level config.lua exposing the options you're expected to touch — framework choice, locale strings, item names, price tables, and similar values. Internal logic is left alone; you shouldn't need to edit anything outside that file for normal use.

Dependencies

Most scripts target either standalone, ESX, or QBCore. Each script page and its bundled README states which frameworks are supported and lists any additional dependency resources (e.g. oxmysql, ox_lib) you'll need to have running first.

Site API

The public site is backed by a small REST API if you want to browse the catalog programmatically:

GET /api/scripts              List all published scripts
GET /api/scripts/:slug        Get a single script's detail
GET /api/categories           List categories
GET /api/scripts/:slug/download   Increments counter, returns file

Support

Found a bug or have a request? Open an issue on the script's linked GitHub repo, or reach out to the crew directly — links are on each script's detail page.
